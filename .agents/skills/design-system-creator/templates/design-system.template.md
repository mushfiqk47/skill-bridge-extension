## Design Guardrails

### Do's
- Use semantic tokens for all values (never hardcode hex/px)
- Follow the spacing scale — no arbitrary values
- Test all components in both light and dark mode
- Include focus states for every interactive element

### Don'ts
- Don't use more than 3 primary colors in one view
- Don't skip heading levels (h1 → h3)
- Don't use opacity for disabled states (use dedicated tokens)
- Don't animate layout properties (width/height) — use transform
- Don't use pure black (#000) as a background in dark mode
