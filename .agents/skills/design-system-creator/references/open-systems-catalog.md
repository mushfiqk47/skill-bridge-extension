## What to Study in Each System

When examining a reference system, look for:

1. **Token Naming** — How do they name their tokens? What hierarchy do they use?
2. **Color Architecture** — How many primitive colors? How are semantic roles defined?
3. **Theming Approach** — How do they handle light/dark? Is it a separate file or CSS variables?
4. **Component States** — How thoroughly are states defined?
5. **Spacing System** — What's the base unit? How many scale values?
6. **Typography Scale** — What ratio? How many sizes?
7. **Documentation Quality** — How do they explain usage and guidelines?
8. **Accessibility** — Is it an afterthought or built into the foundation?
